Historietestsystemet

Tre filer · GitHub Pages · Supabase · Google-inloggning

Systemet består av tre HTML-filer som ligger i ett GitHub-repo och publiceras via GitHub Pages. Inga servrar att driftsätta, inga kostnader. Databasen och inloggningen sköts av Supabase.

Eleverna loggar in med sitt Google-skolkonto, gör ett quiz och ser sitt resultat direkt. Resultaten sparas automatiskt i databasen. Du som lärare ser alla elevers resultat i lärarpanelen — sorterat, filtrerbart och exporterbart till CSV.

Viktigt: Det här systemet är helt skilt från Google Formulär och används enbart för de informella historietesterna. Provresultat i historia hanteras separat via Google Formulär som vanligt.

Elevernas flöde

1
Öppnar länken

Du delar länken till index.html eller visar en QR-kod på tavlan. Eleverna öppnar den i mobilen eller datorn.

2
Loggar in med Google

De klickar på "Logga in med Google" och väljer sitt skolkonto. Sker bara första gången — sedan sker inloggningen automatiskt.

3
Väljer ett test

Listan visar alla tillgängliga quiz. De kan göra om tester hur många gånger de vill — varje försök sparas separat.

4
Svarar på frågorna

En fråga i taget. Efter varje svar ser de direkt om de hade rätt (grönt) eller fel (rött) och vilket alternativ som var korrekt.

5
Ser resultatet

Poäng, procent och genomgång av alla svar visas. Resultatet sparas automatiskt — eleven behöver inte göra något.

Lärarens flöde

1
Öppna lärarpanelen

Gå till lararpanel.html och logga in med ditt Google-konto.

2
Se översikten

Överst visas antal elever, genomförda tester, snittresultat och hur många som gjort test idag. Under det bryts resultaten ned per quiz.

3
Klicka på en elev

Tabellen visar en rad per elev med snittresultat. Klicka på en rad för att expandera och se alla enskilda försök.

4
Filtrera och exportera

Sök på namn, filtrera på vilket quiz som helst, eller exportera allt till en CSV-fil som öppnas i Excel eller Sheets.

Lägg till ett nytt quiz

All quizdata finns i index.html, i en array som heter QUIZZES. Varje quiz är ett objekt med tre fält:

FältVad det är
idUnikt kort namn utan mellanslag, används internt. T.ex. medeltid-del1
nameDet namn som visas för eleverna och i lärarpanelen
questionsEn lista med frågor (se format nedan)

Varje fråga har tre fält

FältVad det är
qFrågetexten
optsEn lista med exakt fyra svarsalternativ (strängar)
correctIndex (0–3) för det rätta svaret. 0 = första alternativet, 1 = andra, osv.

Exempel – så här lägger du till ett nytt quiz

Öppna index.html i en textredigerare. Hitta raden // ── Lägg till fler quiz-objekt här ── och klistra in ditt nya quiz precis före den raden:

// Nytt quiz – Medeltiden
{
  id: 'medeltiden-del1',
  name: 'Medeltiden – Del 1',
  questions: [
    {
      q: 'Vilket år brukar man säga att medeltiden börjar?',
      opts: [
        '476 e.Kr., när Västroms rike föll',
        '1066, efter slaget vid Hastings',
        '313 e.Kr., när kristendomen legaliserades',
        '800 e.Kr., när Karl den store kröntes'
      ],
      correct: 0   // 0 = första alternativet är rätt
    },
    // ... fler frågor ...
  ]
},
Tips: Blanda gärna rätt svars position — låt inte alla rätta svar vara A. Ändra bara värdet på correct (0, 1, 2 eller 3) så pekar systemet alltid på rätt alternativ oavsett ordning.

Spara filen och ladda upp den till GitHub. Quizet syns direkt för eleverna utan att något annat behöver ändras.

Supabase-databasen

Projektet heter Hi-tester, Forntid till 1789 och finns på supabase.com/dashboard. Databasen har tre tabeller:

TabellInnehåll
profilesEn rad per elev med namn och e-post. Skapas automatiskt vid första inloggningen.
quiz_sessionsEtt tillfälle per gång en elev slutför ett quiz — quiz-namn, poäng och tidsstämpel.
quiz_answersVarje enskilt svar: frågetext, valt alternativ, rätt alternativ och om det var rätt.

Se data direkt i Supabase

1
Gå till Table Editor

Supabase Dashboard → ditt projekt → Table Editor i vänstermenyn.

2
Välj quiz_sessions

Här ser du alla genomförda tester med elev-id, poäng och tidpunkt. Klicka på kolumnrubriker för att sortera.

3
Exportera härifrån också

Supabase har en inbyggd CSV-export per tabell via knappen uppe till höger om tabellen.

GitHub Pages

Repot publiceras automatiskt via GitHub Pages. När du uppdaterar en fil (t.ex. lägger till ett nytt quiz i index.html) gör du så här:

1
Öppna repot på GitHub

Gå till github.com/Erengisle/<repo-namn>

2
Klicka på filen du vill ändra

T.ex. index.html → klicka på pennikonen för att redigera direkt i webbläsaren.

3
Spara (Commit changes)

Skriv en kort beskrivning av vad du ändrade, klicka på "Commit changes". GitHub Pages uppdateras automatiskt inom en minut.

Felsökning

Eleven kan inte logga in

Kontrollera att GitHub Pages-URL:en är tillagd under Authentication → URL Configuration i Supabase (både Site URL och Redirect URLs).

Resultaten sparas inte

Eleven måste vara inloggad. Om det fortfarande inte fungerar — kontrollera att tabellerna i Supabase har Row Level Security aktiverat med rätt policies (skapades automatiskt vid setup).

Lärarpanelen visar inga elever

Panelen visar bara elever som faktiskt loggat in och gjort minst ett test. Det är normalt att den är tom i början.

Ett quiz syns inte för eleverna

Kontrollera att du sparat filen och att GitHub Pages hunnit uppdateras (kan ta upp till 1 minut). Kontrollera också att quizobjektet i QUIZZES-arrayen har ett kommatecken efter sig om det inte är det sista i listan.
